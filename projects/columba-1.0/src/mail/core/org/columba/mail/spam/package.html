Tightly integrated Spam Filter using Macchiato library.

Users enables spam filter on a per-account basis in the account options dialog. {@link SpamItem} keeps all options.

Several commands where modified to plug the spam filter into Columba, without adding additional actions. The user only has to mark messages as spam or not spam to train the spam filter. Additionally to {@link MarkMessageCommand}, {@link AddPOP3MessageCommand} was modified to score new messages downloaded from POP3 servers.

Spam token database is saved in /mail/spam.db.
