XML-based configuration management. All config files are registered at {@link Config}.

{@link MailConfig} and {@link AddressbookConfig} are just helper classes which register their configuration files at {@link Config}. This can be seen in the configuration directory structure of the user's configuration directory. Where mail and addressbook have their own folders for storing preferences.

Example on how to get a {@link XmlElement}xml treenode: 

  
  XmlElement gui = MainInterface.config.get("options").getElement("/options/gui");

This would address the file options.xml . Following a little example on how this file would look like: 

  
   <?xml version="1.0" encoding="UTF-8"?>
    <options>
     <gui enabled="true">
      .. your options here
     </gui>
    </options>
